## ★ MX 담당이 반드시 외울 2가지 (CE와 다른 점)

CE(가전)와 MX(모바일)는 로직이 거의 같고, **딱 아래 2개만** 다릅니다.

### (1) 파손보험은 비과세 → v+ 계산이 다름
```
[CE]  총구독료(v+) = 총구독료(v-) × 1.1
[MX]  총구독료(v+) = (제품 + 보증 + 케어) × 1.1  +  파손보험
                     └─── 과세분만 ×1.1 ───┘     └ 비과세 그대로 ┘
```
### (2) 할인 기준금액은 "파손보험 제외"
```
할인기준(v+) = 총구독료(v+) − 파손보험    (파손보험엔 할인 0%)
```

## Lv.1 — 할인 시뮬레이터 (③)  ※여기부터 손으로

예제: 갤럭시탭 SM-X920, **4년(48개월)**, 총구독료(v-) 2,011,200 / 파손보험 115,200 / 보증 96,000 / 케어 0

### 1-A. 기본 구독료
| 항목 | 셀 | 수식 | 값 |
|---|---|---|---|
| 제품 매출인식 기준(v-) | C15 | `총구독료 − 파손 − 보증 − 케어` | 1,800,000 |
| 월구독료(v-) | C17 | `총구독료 ÷ 48` | 41,900 |
| **총구독료(v+) ★** | C18 | `(제품+보증+케어)×1.1 + 파손` | 2,200,800 |
| 할인기준(파손제외 v+) | C20 | `총구독료(v+) − 파손` | 2,085,600 |

### 1-B. 할인 6단계 패턴 (선납할인·단품·결합 전부 동일)
```
① 할인금(v+)   = 총구독료(v+) × 할인율
② 할인금(v-)   = ① ÷ 1.1
③ 월할인금(v-) = ROUNDUP(② ÷ 48, -2)   ← 월로 쪼개 100원 올림
④ 월할인금(v+) = ③ × 1.1
⑤ 최종할인금   = ③(또는 ④) × 48        ← 다시 ×기간 = 깔끔한 총액
⑥ 반영 월구독료 = 월구독료 − 월할인금
```
> 왜 빙 도나? `총액×율`을 바로 월로 나누면 끝자리가 지저분 → **월 단위 먼저 반올림 후 ×48**로 매월 청구액을 딱 떨어지게 만드는 트릭.

| 할인(율) | 월할인금(v-) | 최종(v-) |
|---|---|---|
| 선납할인 4% | 1,700 | 81,600 |
| 단품 5% | 2,100 | 100,800 |
| 결합 5% | 2,100 | 100,800 |

### 1-C. 선납금(≠할인) — 미리 낸 돈
| 항목 | 셀 | 값 |
|---|---|---|
| 선납금%·선납금(v+) | C22·C23 | 30% · 660,240 |
| 월 선납금(v-) = `ROUND(선납금v-/48,-2)` | C25 | 12,500 |
| 최종 선납금(v-) | C26 | 600,000 |

### 1-D. 전체 합산 & 49회차 청구표
```
전체할인 반영 월구독료(v-) C61
 = (총구독료 − 선납금 − 단품 − 선납할인 − 결합) ÷ 48
 = (2,011,200 − 600,000 − 100,800 − 81,600 − 100,800) ÷ 48 = 23,500
```
**회차표 규칙 (총 49회차 = 계약 48개월 + 1)**
| 규칙 | 수식 |
|---|---|
| 1회차(일할) | `ROUNDDOWN((총액/48) × 사용일수/그달일수, -2)` → 26,500 |
| 2~48회차(풀) | `= 월구독료` → 41,900 |
| **49회차(막달)** | `월구독료 − 1회차금액` = 41,900 − 26,500 = **15,400** |
| 월청구(v+) | `(제품+케어+보증)×1.1 + 파손` |
| 청구일 | `월말 + 15일` (익월 15일) |

> ✅ 검증: 1회차 + 49회차 = 한 달치. J열 합계 = 2,011,200 (총구독료와 일치)

## Lv.2 — 프라이싱 엔진 (① A부)  RRP → 월구독료

예제: 갤럭시탭 SM-X820, **4년**, RRP 1,399,200 / 혜택가(v+) 1,344,000

### 2-A. 입력 (외부 시트 VLOOKUP)
| 항목 | 출처 | 값 |
|---|---|---|
| RRP / 혜택가(v+) | `RRP,CGS` 시트 | 1,399,200 / 1,344,000 |
| 보증 판가/원가, 파손보험 | `케어보증 Master` | 96,000 / 9,600 / 115,200 |
| CGS(연결원가) | `RRP,CGS` 시트 | 690,074 |

### 2-B. 혜택가(v-) → 비용 요소 (전부 혜택가 비례)
```
혜택가(v-) AL = ROUND(혜택가v+ / 1.1) = 1,221,818

이자비용 AU   = AL − PV(4%/12, 48, −AL/48) = 94,466   ★금융리스 이자
거래선수수료 AW = AL × 12.6% = 153,949   (삼판; 닷컴이면 0)
인프라 BA      = AL × 0.5%  = 6,109
중도해지Risk BB = AL × 4.1%  = 50,094
구독포인트 BD   = AL × 9.66% = 118,028
보험료 BE      = (위 합계) × 0.553% × 4년 = 39,384
판매인센/시상   = 30,000 / 10,000 (절대값)
```

### 2-C. 합산 → 월구독료 → 정수처리
```
구독료총합(v-) BI = SUM(혜택가, 보증, 인센, 시상, 인프라, 중도해지Risk,
                        결합할인, 구독포인트, 이자비용, 거래선수수료, 보험료)
               = 1,819,848
순수월할 BO  = ROUND((BI − 케어 − 보증) / 48, -2) = 35,900
월구독료(v-) BS = 순수월할 + 케어월할 + 보증월할 = 35,900 + 0 + 2,000 = 37,900
총구독료(v-) BR = 37,900 × 48 = 1,819,200   ← ★ Lv.1의 입력값이 됨
```
> 핵심 통찰: **월구독료 = (혜택가 + 모든 비용·포인트·이자·수수료) ÷ 개월.**
> 즉 "원가가 아니라 **비용을 다 얹은 가격**"이고, 비용률(주요비용률 시트)이 프라이싱의 다이얼이다.

## Lv.3 — 손익 엔진 (① B부)  금융리스 회계

> **핵심 개념: 구독 = 제품을 리스(할부판매)한 것.**
> → 설치월에 **제품매출·원가(CGS)를 즉시 인식**, 나머지 구독료는 **이자수익**으로 기간 배분.

```
구독매출인식액(A) DB = MIN(순수구독료 PV, 혜택가v-) = 1,221,818   (= 제품을 판 매출)
이자수익(D) DC      = 매출인식구독료 − DB          = 597,382      (기간 배분)
CGS 매출원가 DJ     = RRP,CGS 시트               = 690,074
원가율 DK          = CGS / 혜택가v-              = 60%
이자손익 DX        = 이자수익 − 실제조달이자(2.9%) = 527,866
제품이익(설치월) DP = 매출인식 + 포인트차감 − CGS − O/H + 충당 = −216,925  (설치월 적자)
제품이익율(구독기간) DU = (DP + 이자손익) / 매출 = +20%   (이자로 흑자 전환)
BEP EB = ROUNDUP((CGS + O/H − 포인트차감) × 48 / 매출인식구독료) = 38개월차
```
> 즉 **제품만 보면 설치월 적자지만, 금융리스 이자수익으로 구독기간 통산 흑자**.
> BEP 38개월 → 48개월 계약 내 도달.

### 3-B. 연차별 해지율 채권상각 (FN~NK열, 200컬럼+)
1~6년차 각각에 **해지율**을 적용 → 제품/케어/보증/이자 매출·손익을 다시 계산하고,
`미상각 채권잔액 / 충당금 / 해지 위약금 / 선납 잔액 환불`로 해지손실을 상각.
→ 이 부분은 해지율 테이블(`주요비용률` 시트 D28~) 연동이 필요. **Lv.3 후반 과제.**
