## 1. 왜 "리스 상각표"인가 — 개념

**구독 = 제품을 할부(금융리스)로 판 것.** 회사는 제품을 고객에게 넘기고(채권 발생), 매달 구독료(=리스료)로 회수한다. 회계상:
- 제품을 넘긴 시점에 **리스채권(받을 돈)** 이 잡힌다 = 제품의 v- 값
- 매달 받는 구독료 안에는 **원금 회수분 + 이자수익**이 섞여 있다
- 시간이 갈수록 채권 잔액이 줄고(상각), 그에 비례해 이자수익도 줄어든다
- 만기에 채권 = 0

> 핵심 직관: **"제품값을 빌려주고 이자 붙여 나눠 받는 표"**. 그래서 할인으로 받을 돈이 줄면 → **내재이자율(IRR)을 다시 계산**해야 표가 만기 0으로 떨어진다.

## 3. 엔진 4대 수식 (★여기가 전부)

```
① 리스채권  C2 = MIN( 제품v-,  PV(현할차/12, 60, −순수월액) )
              = MIN(4,809,091,  PV(3.57%/12,60,−116,000))
              = MIN(4,809,091, 6,365,543) = 4,809,091
   · 순수월액 = (총구독료 − 케어 − 보증)/60 = 6,960,000/60 = 116,000
   · 제품v-가 더 작으니 그게 채권(=받을 원금)

② 총 순수구독료 C3 = 총구독료 − 케어 − 보증 = 6,960,000
③ 리스료      H2 = C3 / 60 = 116,000      (월 회수액, 순수 제품포션)

④ 월이자율(IRR) F3 = RATE(60, −116,000, 4,809,091) = 0.01302355  (1.302%/월)
   연이자율   F2 = F3 × 12 = 15.628%
   → "채권 4,809,091을 매달 116,000씩 60번 받아 0으로 만드는 내재이자율"
```

> **RATE(엑셀)** = 내부수익률(IRR). 받을 원금(현재가치)·매월 받는 돈·기간을 주면, 그걸 정확히 상환시키는 이자율을 역산. 이게 엔진의 심장.

## 4. 상각표 본문 (매월 한 줄)

열: **C 기초 → D 이자수익 → E 리스료수령(−) → F 기말**, `기말 = 기초 + 이자 − 리스료`

| 회차 | 기초 C | 이자 D | 리스료 E | 기말 F |
|---|---|---|---|---|
| 0 (설치) | | | | 4,809,091 |
| **1 (일할 27/31)** | 4,809,091 | 54,618 | −101,100 | 4,762,609 |
| 2 | 4,762,609 | 62,026 | −116,000 | 4,708,635 |
| 3 | 4,708,635 | 61,323 | −116,000 | 4,653,958 |
| … | | | | |
| 60 | … | | −116,000 | 14,775 |
| 61 (정리) | 14,775 | +125 | −14,900 | **0** |

**행별 수식**
```
C행 = 직전 F                         (기초 = 전월 기말)
D행 = ROUND(C × F3)                  (이자 = 기초 × 월이자율)
E행 = −H2 = −116,000                 (리스료, 매월 고정)
F행 = C + D + E                      (기말)

[1회차만 일할]
E6 = −(ROUNDDOWN(총구독/60×27/31,−2) − 케어일할 − 보증일할) = −101,100
D6 = ROUND(−E6 + (C2×F3×27/31 − H2×27/31)) = 54,618

[마지막 회차] 잔액을 이자/리스료로 흡수해 F=0 (plug)
```

> ✅ 검증: 1회차 기말 4,762,609, …, 60회차 후 14,775 → 61회차에 0. (정답지 `engine_amort.py`와 일치)

## 5. 시트18 — 선납 반영 (엔진이 어떻게 바뀌나)

선납금(고객이 미리 낸 목돈)이 들어오면 **받을 채권이 줄고 → 리스료가 줄고 → IRR을 다시 푼다.**

```
선납금(최종 V-) S7 = 744,000          (총구독료의 약 10% 예시)
① 리스채권 C2 = MIN(…) − S7 = 4,809,091 − 744,000 = 4,065,091
② 순수구독료 C3 = 6,960,000 − 744,000 = 6,216,000
③ 리스료 H2 = 6,216,000 / 60 = 103,600   (116,000 → 103,600 ↓)
④ 월이자율 F3 = RATE(60, −103,600, 4,065,091) = 0.01514  → 연 18.17%
```

| | 선납前(17) | 선납後(18) |
|---|---|---|
| 리스채권 | 4,809,091 | **4,065,091** |
| 리스료/월 | 116,000 | **103,600** |
| 연이자율 | 15.63% | **18.17%** |

> 포인트: 선납은 **채권·리스료를 줄이지만 IRR은 올라간다**(남은 금액을 더 짧게 굴리는 효과). 숫자 자체보다 **"입력이 바뀌면 IRR을 반드시 재계산"** 이라는 구조를 체득할 것.

## ✍️ 직접 작성 연습 (빈 엑셀)
1. 입력 7칸(N1,K2,S2,S3,S5,S6) 적기
2. C2,C3,H2,F3,F2 **4대 수식**만으로 채권·리스료·이자율 산출 → 본문 값과 대조
3. 상각표 C~F 4열을 1행 만들고 **아래로 복사** → 60행
4. 1회차 일할, 마지막 plug 처리
5. 선납금 744,000을 넣어 C2,C3,H2,F3가 바뀌는지 확인

## ✅ 체크포인트 (답 가리고)
1. 리스채권이 MIN(제품v-, PV…)인 이유는? → *제품을 판 것이므로 받을 원금은 제품값을 넘을 수 없음*
2. 리스료에 케어·보증을 왜 뺐나? → *케어·보증은 리스(금융)가 아니라 서비스 매출이라 채권 상각과 분리*
3. F3는 무슨 함수로 구하나? → *RATE = IRR*
4. 선납 744,000 → 리스료는? → *103,600*
5. 마지막 회차 잔액이 정확히 0이 되게 하는 처리 이름은? → *plug(보정행)*

## ⚠️ 함정
- **이자율을 고정값으로 착각** 금지 → 입력(할인·선납) 바뀌면 **매번 RATE 재계산**
- 1회차는 **반드시 일할**(27/31). 통째로 넣으면 표 전체가 어긋남
- 케어·보증을 리스료에 섞으면 채권이 안 맞음
- ROUND/ROUNDDOWN 자리(−2 = 100원) 틀리면 끝자리 누적 오차
